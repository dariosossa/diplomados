¿Cómo colocar listas en HTML?

En HTML se pueden crear listas para organizar información.

Existen principalmente dos tipos de listas: listas ordenadas y listas no ordenadas.

Los elementos de la lista se crean con la etiqueta li.

Lista no ordenada (con viñetas): 


<ul>
  <li>Manzana</li>
  <li>Naranja</li>
  <li>Banana</li>
</ul>

Lista ordenada (con números): 


<ol>
  <li>Primer paso</li>
  <li>Segundo paso</li>
  <li>Tercer paso</li>
</ol>

La lista ordenada permite cambiar el estilo de numeración utilizando el atributo type. 


<ol type="A">
  <li>Primer punto</li>
  <li>Segundo punto</li>
  <li>Tercer punto</li>
</ol>

Tipos más comunes:

  • 1 → números (por defecto)
  • A → letras mayúsculas
  • a → letras minúsculas
  • I → números romanos mayúsculos
  • i → números romanos minúsculos

Se utiliza el atributo start para indicar desde qué número comenzará la lista. 


<ol start="5">
  <li>Paso cinco</li>
  <li>Paso seis</li>
  <li>Paso siete</li>
</ol>

Se utiliza el atributo reversed para mostrar la lista en orden descendente. 


<ol reversed>
  <li>Tercer lugar</li>
  <li>Segundo lugar</li>
  <li>Primer lugar</li>
</ol>

La Era de las Máquinas y el Vapor

Del Taller Artesanal a la Fábrica Moderna

La Revolución Industrial, iniciada en Gran Bretaña a finales del siglo XVIII, no fue solo un cambio tecnológico, sino una transformación total de la existencia humana. La invención de la máquina de vapor por James Watt permitió mecanizar procesos que antes dependían de la fuerza animal o humana.

Innovaciones Técnicas de la Primera Etapa

El despegue industrial se apoyó en tres pilares fundamentales que permitieron un crecimiento económico sin precedentes:

  • El Carbón: La fuente de energía primaria que alimentaba las calderas.
  • El Hierro: Material esencial para la construcción de maquinaria y ferrocarriles.
  • La Industria Textil: El primer sector en adoptar la producción en masa.

Evolución de los Medios de Transporte

La necesidad de mover mercancías rápidamente dio lugar a una revolución en la ingeniería civil:

  1. Construcción de canales navegables para el transporte de carbón a bajo costo.
  2. Desarrollo de la Locomotora de Vapor (The Rocket) por George Stephenson.
  3. Aparición de los primeros barcos de vapor transatlánticos.

Consecuencias Sociales a Largo Plazo

No todo fue progreso técnico; la sociedad tuvo que adaptarse a nuevas y a veces duras realidades laborales:

  1. Surgimiento del proletariado urbano y los sindicatos.
  2. Regulación de las jornadas de trabajo y prohibición del trabajo infantil.
  3. Crecimiento exponencial de la clase media comercial.

Hitos de la Segunda Revolución Industrial

Hacia finales del siglo XIX, la electricidad y el petróleo tomaron el relevo:

  1. Invención de la bombilla incandescente por Thomas Edison.
  2. Producción en cadena implementada por Henry Ford en el sector automotriz.
  3. Desarrollo de la industria química y los primeros plásticos sintéticos.

El Cerebro: La Frontera Final del Conocimiento

La Red de Conexiones más Compleja del Universo

El cerebro humano pesa apenas 1.4 kilogramos, pero contiene aproximadamente 86.000 millones de neuronas. La neuroplasticidad es la capacidad de este órgano para reorganizarse a sí mismo, creando nuevas rutas de aprendizaje a lo largo de toda la vida.

Estructura y Funciones de los Lóbulos Cerebrales

Para facilitar su estudio, la corteza cerebral se divide en cuatro áreas principales:

  1. Lóbulo Frontal: Sede del razonamiento, la planificación y el control emocional.
  2. Lóbulo Parietal: Encargado de procesar información sensorial como el tacto y el dolor.
  3. Lóbulo Temporal: Fundamental para la memoria y el procesamiento del lenguaje.
  4. Lóbulo Occipital: Dedicado casi exclusivamente a la interpretación de la visión.

El Proceso de la Sinapsis

La comunicación entre neuronas ocurre en un espacio microscópico llamado espacio sináptico:

  1. Llegada del impulso eléctrico al terminal de la neurona emisora.
  2. Liberación de neurotransmisores (como la dopamina o serotonina).
  3. Recepción de las moléculas por los receptores de la neurona receptora.
  4. Generación de un nuevo impulso eléctrico en la célula destino.

Hábitos para un Cerebro Saludable

  • Actividad Física: Mejora la oxigenación y la producción de factores neurotróficos.
  • Sueño Reparador: Durante el descanso, el cerebro elimina toxinas metabólicas.
  • Aprendizaje de Idiomas: Aumenta la densidad de la materia gris y retrasa el declive cognitivo.

Curiosidades de la Neurociencia

  1. El cerebro consume el 20% de la energía total del cuerpo humano.
  2. No existen receptores de dolor en el tejido cerebral.
  3. La información viaja a velocidades de hasta 400 km/h.

¿Qué es id y class en HTML?

En HTML podemos identificar elementos para aplicar estilos específicos utilizando los atributos id y class.

Estos atributos permiten que CSS seleccione ciertos elementos y les aplique estilos.

Uso de id

El atributo id se utiliza para identificar un elemento único dentro de la página. No debe repetirse. 


<h1 id="titulo-principal">
    Título principal
</h1>

En CSS el id se selecciona utilizando el símbolo #. 


#titulo-principal {
  border: 4px solid red;
}

Esto aplicará el estilo únicamente al elemento que tenga ese id.

Uso de class

El atributo class se utiliza cuando queremos aplicar el mismo estilo a varios elementos. 


<h2 class="titulo">
    Primer título
</h2>

<h3 class="titulo">
    Segundo título
</h3>

En CSS la class se selecciona utilizando un punto . 


.titulo {
  border: 3px solid blue;
}

Esto aplicará el mismo estilo a todos los elementos que tengan esa class.

Diferencia entre id y class

  • id → identifica un solo elemento.
  • class → puede aplicarse a varios elementos.

¿Cómo colocar una imagen como fondo en CSS?

En CSS podemos colocar una imagen como fondo de un elemento utilizando la propiedad background-image.

Esta propiedad permite mostrar una imagen detrás del contenido de un elemento como un div, un título o una sección.

Ejemplo usando class

<div class="banner">
  Contenido del banner
</div>

.banner {
  background-image: url("img/fondo.jpg");
}

La función url() indica la ubicación de la imagen dentro del proyecto.

En este ejemplo la imagen se encuentra dentro de la carpeta img.

Configuración para que la imagen se vea mejor

Cuando colocamos una imagen de fondo, es recomendable ajustar algunas propiedades para mejorar su apariencia.

Las más utilizadas son:

  • background-size → controla el tamaño de la imagen
  • background-position → controla la posición de la imagen
  • background-repeat → controla si la imagen se repite

Ejemplo completo


.banner {
  background-image: url("img/fondo.jpg");
  background-size: cover;
  background-position: center;
  background-repeat: no-repeat;
}

En este ejemplo:

  • background-size: cover → la imagen cubre todo el elemento
  • background-position: center → la imagen se centra
  • background-repeat: no-repeat → evita que la imagen se repita

Esta configuración se utiliza con frecuencia en diseño web moderno para crear secciones con imágenes de fondo limpias y bien ajustadas.

¿Qué es la propiedad display en CSS?

La propiedad display controla la forma en que un elemento se comporta dentro de la página. Es decir, define cómo se muestra y cómo ocupa el espacio.

Dependiendo del valor que utilicemos, el elemento puede ocupar toda la línea, compartir espacio con otros elementos o incluso desaparecer visualmente.

Algunos de los valores más utilizados son:

  • display: block
  • display: inline
  • display: inline-block
  • display: none

¿Qué significa display: block?

Cuando un elemento tiene display: block, ocupa todo el ancho disponible de la página y provoca un salto de línea antes y después del elemento.

Esto significa que ningún otro elemento puede colocarse en la misma línea.

Algunos elementos HTML ya son block por defecto, como:

  • div
  • p
  • h1, h2, h3...

.elemento {
  display: block;
}

Esto hace que el elemento se comporte como una caja que ocupa toda la línea.

¿Qué significa display: inline?

Cuando utilizamos display: inline, el elemento solo ocupa el espacio necesario para su contenido.

Esto permite que varios elementos se coloquen uno al lado del otro en la misma línea.

Algunos elementos HTML que ya son inline por defecto son:

  • span
  • a
  • strong

.elemento {
  display: inline;
}

Un detalle importante es que los elementos inline no respetan bien las propiedades de width y height.

¿Qué significa display: inline-block?

El valor inline-block combina características de inline y block.

Esto significa que:

  • El elemento puede colocarse al lado de otros elementos.
  • Pero también permite usar width, height, padding y margin.

Este comportamiento es muy útil cuando queremos crear elementos alineados horizontalmente, como botones o tarjetas. 


.elemento {
  display: inline-block;
  width: 150px;
  height: 80px;
}

¿Qué significa display: none?

Cuando utilizamos display: none, el elemento desaparece completamente de la página.

El navegador actúa como si ese elemento no existiera.

Esto es diferente de simplemente ocultarlo visualmente, porque el espacio que ocupaba también desaparece. 


.elemento {
  display: none;
}

Este valor se utiliza mucho en programación web para ocultar o mostrar elementos dinámicamente con JavaScript.

¿Qué es Flexbox en CSS?

Flexbox es un sistema de diseño en CSS que permite organizar elementos de forma flexible dentro de un contenedor. Su objetivo es facilitar la distribución y alineación de los elementos dentro de una fila o columna.

Para utilizar Flexbox primero debemos convertir un elemento en contenedor flexible. Esto se hace utilizando la propiedad display: flex.

Cuando un elemento se convierte en contenedor flexible, todos los elementos que están dentro de él pasan a comportarse como elementos flexibles.

¿Qué hace display: flex?

La propiedad display: flex convierte un elemento en un contenedor flexible. Esto permite controlar fácilmente la posición y distribución de los elementos hijos.

Por defecto, cuando usamos display: flex, los elementos se colocan uno al lado del otro en forma horizontal. 


.contenedor {
  display: flex;
}

HTML donde se aplicaría este estilo: 


<div class="contenedor">
  <div>Caja 1</div>
  <div>Caja 2</div>
  <div>Caja 3</div>
</div>

En este caso las tres cajas aparecerán alineadas en una misma fila.

¿Cómo distribuir el espacio entre los elementos?

Para controlar cómo se distribuyen los elementos dentro del contenedor se utiliza la propiedad justify-content.

Esta propiedad controla la alineación de los elementos en el eje horizontal. 


.contenedor {
  display: flex;
  justify-content: center;
}

Esto centrará los elementos horizontalmente dentro del contenedor.

Algunos valores comunes de justify-content son:

  • flex-start → elementos alineados al inicio
  • center → elementos centrados
  • flex-end → elementos alineados al final
  • space-between → espacio entre los elementos
  • space-around → espacio alrededor de los elementos

En las interfaces modernas es común encontrar más de un botón dentro de una misma sección. Generalmente se utiliza un botón principal para la acción más importante y un botón secundario para una acción alternativa.

Por ejemplo, en un formulario podemos tener un botón Guardar para confirmar la información y un botón Cancelar para salir del proceso sin realizar cambios.

El botón principal debe llamar más la atención, mientras que el botón secundario debe verse más neutral para no competir visualmente con la acción principal.

Paso 1. Crear la estructura HTML

Primero se crea un contenedor que agrupará ambos botones dentro de la misma sección. 


<div class="acciones">

  <button class="btn-guardar">Guardar</button>
  <button class="btn-cancelar">Cancelar</button>

</div>

El contenedor acciones permite organizar los botones dentro de un mismo espacio de la interfaz.

Paso 2. Organizar los botones con Flexbox

Para que los botones se alineen correctamente se puede utilizar Flexbox. Esto permite colocarlos uno al lado del otro y generar espacio entre ellos. 


.acciones{
  display:flex;
  gap:10px;
}

La propiedad gap crea un espacio entre los botones para evitar que queden demasiado juntos.

Paso 3. Diseñar el botón principal (Guardar)

El botón principal debe tener un color más fuerte para destacar la acción principal que realizará el usuario. 


.btn-guardar{
  background-color:#2563eb;
  color:white;
  padding:10px 20px;
  border:none;
  border-radius:6px;
  cursor:pointer;
}

Este botón utiliza un color azul porque generalmente se asocia con acciones positivas como confirmar o guardar información.

Paso 4. Diseñar el botón secundario (Cancelar)

El botón cancelar debe ser visualmente más neutral para indicar que no es la acción principal del proceso. 


.btn-cancelar{
  background-color:white;
  color:#374151;
  padding:10px 20px;
  border:1px solid #d1d5db;
  border-radius:6px;
  cursor:pointer;
}

Este estilo genera un botón más discreto que acompaña al botón principal sin competir visualmente con él.

Paso 5. Agregar estados hover

Los estados hover ayudan a indicar que el botón puede interactuar con el usuario cuando el cursor pasa sobre él. 


.btn-guardar:hover{
  background-color:#1d4ed8;
}

.btn-cancelar:hover{
  background-color:#f3f4f6;
}

Paso 6. Agregar estado activo

El estado active genera una pequeña animación cuando el usuario presiona el botón, simulando que el botón se hunde. 


.btn-guardar:active,
.btn-cancelar:active{

¿Tailwind también utiliza el modelo de caja?

Sí. Tailwind también utiliza el modelo de caja (Box Model), porque Tailwind no reemplaza a CSS. En realidad es un framework que utiliza las mismas reglas de CSS que el navegador ya conoce.

Esto significa que todos los elementos de una página siguen comportándose como cajas rectangulares formadas por cuatro partes: margin, border, padding y content.

Significado de los números:

¿Cómo cambiar el comportamiento de display en Tailwind?

En HTML cada etiqueta tiene un comportamiento por defecto definido por la propiedad display. Algunos elementos ocupan toda la línea y otros solo ocupan el espacio de su contenido.

Por ejemplo:

Elementos block → ocupan toda la línea (div, p, h1, section)

Elementos inline → solo ocupan el espacio del contenido (span, a, b)

Con Tailwind podemos cambiar ese comportamiento usando clases que representan directamente la propiedad display.

Clases display disponibles en Tailwind

block → convierte el elemento en bloque

inline → convierte el elemento en elemento en línea

inline-block → elemento en línea pero permite ancho y alto

1. Convertir un elemento inline en block

La etiqueta span normalmente es inline, pero podemos convertirla en bloque usando la clase block. 


<span class="block bg-blue-200">
Ahora este span se comporta como bloque
</span>

Ahora el span ocupará toda la línea, como si fuera un div.

2. Convertir un elemento block en inline

Un div normalmente ocupa toda la línea. Con Tailwind podemos hacer que se comporte como un elemento en línea. 


<div class="inline bg-green-200">Caja 1</div>
<div class="inline bg-green-300">Caja 2</div>
<div class="inline bg-green-400">Caja 3</div>

Como ahora son inline, los elementos aparecerán uno al lado del otro.

3. Usar inline-block

inline-block permite que el elemento se comporte como inline (aparecer en la misma línea), pero permite controlar su tamaño. 


<div class="inline-block w-32 h-16 bg-purple-300">
Caja A
</div>

<div class="inline-block w-32 h-16 bg-purple-400">
Caja B
</div>

Ambas cajas estarán en la misma línea y tendrán tamaño definido.

¿Qué es JavaScript?

Es el lenguaje de programación que permite crear contenido dinámico e interactivo en las páginas web. Mientras que el HTML estructura y el CSS da estilo, JS controla el comportamiento.

Conexión con HTML

Para que el navegador ejecute tu código, debes vincularlo usando la etiqueta script, preferiblemente antes de cerrar el body.

<!-- Vinculación de archivo externo -->
<script src="app.js"></script>

Variables y Constantes

Son espacios de memoria donde guardamos datos. Las variables pueden cambiar su valor, mientras que las constantes permanecen fijas.

let usuario = "Carlos"; // Variable
const ID = 5502;       // Constante

Let vs Const

Usa "let" cuando sepas que el valor necesita ser reasignado más tarde. Usa "const" por defecto para que tu código sea más seguro y predecible.

let vidas = 3;
vidas = 2; // Permitido

const tipo = "Humano";
tipo = "Robot"; // Error: No se puede reasignar

Tipos de Datos Primitivos

Son los valores más básicos del lenguaje.

// 1. String (Cadenas de texto)
let nombre = "Ana"; 

// 2. Number (Enteros y decimales)
let edad = 25;
let precio = 19.99;

// 3. Boolean (Lógica)
let esMayor = true; // o false

// 4. Undefined (Valor no asignado)
let test; // Su valor es undefined

// 5. Null (Ausencia intencional de valor)
let vacio = null;

Operador typeof

Si alguna vez tienes dudas sobre qué tipo de dato es una variable, puedes usar typeof para descubrirlo. 

console.log(typeof "Hola"); // "string"
console.log(typeof 42);     // "number"
console.log(typeof true);   // "boolean"

Operadores Aritméticos

Permiten realizar cálculos matemáticos con datos de tipo Number
. El símbolo = asigna el resultado a una variable.

let a = 10;
let b = 5;

let suma = a + b;          // 15
let resta = a - b;         // 5
let multiplicacion = a * b; // 50
let division = a / b;       // 2

El caso especial del "+"

¡Cuidado! Si usas el signo + con un String, JavaScript no sumará, sino que "pegará" (concatenará) los textos.

let texto = "Hola " + "Mundo"; // "Hola Mundo"
let mixto = "Edad: " + 25;     // "Edad: 25" (Lo vuelve texto)

Módulo y Exponente

Existen operadores para obtener el residuo de una división o elevar un número a una potencia.

// Módulo (Residuo de la división)
let residuo = 10 % 3;   // 1 (Porque 3x3=9, sobra 1)

// Exponente (Potencia)
let potencia = 2 ** 3;  // 8 (2 al cubo)

Orden de Operaciones

JavaScript resuelve las operaciones en un orden específico. Si quieres que una suma se realice antes que una multiplicación, debes usar paréntesis. 

// Sin paréntesis (Multiplica primero)
let resultado1 = 5 + 2 * 10;   // 25

// Con paréntesis (Suma primero)
let resultado2 = (5 + 2) * 10; // 70

Cálculos Complejos

Puedes anidar paréntesis dentro de otros para organizar fórmulas matemáticas más difíciles. Siempre se resuelve de adentro hacia afuera.

// Promedio de tres notas
let nota1 = 8;
let nota2 = 9;
let nota3 = 10;

let promedio = (nota1 + nota2 + nota3) / 3; // 9

Legibilidad

A veces, aunque el orden matemático sea correcto, usamos paréntesis para que el código sea más fácil de leer para otros humanos.

// Es lo mismo, pero el segundo es más claro
let operacion = 100 + (50 / 2);

Unión de Textos (Concatenación)

La operación más simple es pegar dos cadenas de texto. Se usa el signo +. Si unes un texto con un número, JavaScript convertirá todo a texto.

let saludo = "Hola" + " " + "Mundo"; // "Hola Mundo"
let puntos = "Score: " + 10;          // "Score: 10"

Contar Caracteres (Length)

Para saber qué tan largo es un texto (incluyendo espacios), usamos la propiedad length. Es fundamental para validar contraseñas o formularios.

let clave = "123456";
console.log(clave.length); // 6

let espacio = " "; 
console.log(espacio.length); // 1

Acceso por Índice

Cada letra tiene una posición. ¡Importante! En programación empezamos a contar desde 0. La primera letra es el índice 0, la segunda el 1, y así sucesivamente.

let fruta = "Piña";

console.log(fruta[0]); // "P"
console.log(fruta[1]); // "i"

Cambiar Formato (Métodos)

Podemos transformar el texto a mayúsculas, minúsculas o limpiar espacios accidentales en los extremos. 

let txt = "  Hola  ";

console.log(txt.toUpperCase()); // "  HOLA  "
console.log(txt.toLowerCase()); // "  hola  "
console.log(txt.trim());        // "Hola" (Quita espacios)

Template Literals (Backticks)

Es la forma profesional de mezclar texto y variables. Usamos comillas invertidas ` y metemos las variables dentro de ${ }. Es más limpio que usar el signo +.

let nombre = "Ana";
let edad = 20;

// "Ana tiene 20 años"
let frase = `${nombre} tiene ${edad} años.`;

Búsqueda y Verificación

Operaciones complejas para saber si un texto contiene ciertas palabras o si empieza de una forma específica.

let correo = "contacto@google.com";

console.log(correo.includes("@"));     // true (Contiene arroba)
console.log(correo.startsWith("con")); // true (Empieza con "con")
console.log(correo.endsWith(".es"));   // false (No termina en .es)

Comparar Valores

Los operadores de comparación analizan dos valores y nos devuelven true (verdadero) o false (falso). Son la base de la lógica. 

let a = 10;
let b = 5;

console.log(a > b);  // true  (Mayor que)
console.log(a < b);  // false (Menor que)
console.log(a >= 10); // true  (Mayor o igual)
console.log(b <= 3);  // false (Menor o igual)

Igualdad Estricta (===)

En JS existen dos formas de comparar igualdad. La recomendada es la estricta (===), que compara el valor y también el tipo de dato.

// Igualdad débil (Evitarla)
console.log(5 == "5");  // true (JS ignora el tipo)

// Igualdad estricta (Usar siempre)
console.log(5 === "5"); // false (Porque uno es número y otro texto)

// Diferente de (Desigualdad)
console.log(10 !== 5);  // true

Operador AND (&&)

Devuelve true solo si todas las condiciones se cumplen. Si una sola es falsa, el resultado será false.

let usuario = "admin";
let clave = 1234;

// ¿El nombre es admin Y la clave es 1234?
console.log(usuario === "admin" && clave === 1234); 
// Resultado en consola: true

console.log(usuario === "admin" && clave === 0000); 
// Resultado en consola: false

Operador OR (||)

Devuelve true si al menos una de las condiciones es verdadera. Es ideal para opciones alternativas.

let tieneEfectivo = false;
let tieneTarjeta = true;

// ¿Puede pagar con alguno de los dos?
console.log(tieneEfectivo || tieneTarjeta); 
// Resultado en consola: true

console.log(false || false); 
// Resultado en consola: false

Operador NOT (!)

Invierte el estado lógico. Es como decir "No" o "Lo opuesto a...".

let encendido = true;

console.log(!encendido); 
// Resultado en consola: false

let esAdulto = false;
console.log(!esAdulto); 
// Resultado en consola: true (No es adulto)

Condicional If / Else

Permite ejecutar un bloque de código si se cumple una condición, y otro bloque si no se cumple.

let edad = 18;

if (edad >= 18) {
    console.log("Eres mayor de edad. ¡Bienvenido!");
} else {
    console.log("Eres menor de edad. Acceso restringido.");
}

Else If (Múltiples opciones)

Cuando tienes más de dos caminos posibles, usas else if para evaluar nuevas condiciones en orden.

let hora = 14;

if (hora < 12) {
    console.log("Buenos días");
} else if (hora < 20) {
    console.log("Buenas tardes");
} else {
    console.log("Buenas noches");
}

Operador Ternario

Es una forma abreviada de escribir un if / else en una sola línea. Muy útil para asignaciones rápidas.

Estructura: condición ? true : false

let puntos = 90;

// Si puntos >= 90, es "Nivel Pro", si no, es "Nivel Novato"
let rango = puntos >= 90 ? "Nivel Pro" : "Nivel Novato";

console.log(rango); // "Nivel Pro"

Bucle For

Se usa cuando sabes exactamente cuántas veces quieres repetir una acción. Se compone de: inicio; condición; incremento.

// Contar del 1 al 5
for (let i = 1; i <= 5; i++) {
    console.log("Vuelta número: " + i);
}

Bucle While

Ejecuta el código mientras la condición sea verdadera. ¡Cuidado! Si la condición nunca cambia a falsa, el bucle será infinito.

let bateria = 3;

while (bateria > 0) {
    console.log("App funcionando... Batería: " + bateria + "%");
    bateria--; // Restamos para que eventualmente termine
}
console.log("Teléfono apagado");

Bucle Do While

La diferencia clave es que este bucle se ejecuta siempre al menos una vez, y después revisa si debe continuar.

let intentos = 0;

do {
    console.log("Intentando conectar...");
    intentos++;
} while (intentos < 1); 
// Aunque la condición sea falsa pronto, entró una vez.

Anatomía de una Función

function cocinar(ingrediente) {
    let plato = "Sopa de " + ingrediente;
    return plato;
}
  • Palabra reservada function: Indica al navegador que estamos definiendo una función.
  • Nombre (cocinar): Es el identificador que usaremos para ejecutar la función más tarde. Se recomienda usar verbos.
  • Paréntesis (): Aquí dentro van los datos que la función necesita para trabajar (llamados parámetros). Si no necesita nada, se dejan vacíos.
  • Llaves {}: Delimitan el "cuerpo" de la función. Todo lo que esté adentro es el código que se ejecutará.
  • Palabra reservada return: Es la salida. Especifica qué valor debe devolver la función al terminar.

Parámetros y Argumentos

Aunque parecen lo mismo, tienen una diferencia clave de concepto:

// El 'parámetro' es el nombre en la definición (la variable vacía)
function saludar(nombre) { 
    return "Hola " + nombre;
}

// El 'argumento' es el valor real que enviamos al llamarla
saludar("Andrea"); // "Andrea" es el argumento

La Llamada (Ejecución)

Definir una función no la ejecuta. Para que el código dentro de las llaves funcione, debes "llamar" a la función usando su nombre seguido de paréntesis.

function aviso() {
    console.log("¡Cuidado!");
}

aviso(); // Aquí es donde realmente se ejecuta el console.log

Funciones de Flecha (Arrow Functions)

Es una forma moderna y simplificada de escribir funciones. Se eliminan las palabras function y return (en ciertos casos) para usar el símbolo de la flecha =>. 

// Función clásica
function triple(n) {
    return n * 3;
}

// Función flecha (Equivalente)
const tripleFlecha = (n) => n * 3;

Elementos de la Función Flecha

Al ser una versión resumida, sus componentes cambian ligeramente para ser más directos:

  • Constante (const): A diferencia de las clásicas, las funciones flecha suelen guardarse en una variable o constante.
  • Paréntesis (): Siguen guardando los parámetros.
  • La Flecha =>: Conecta los parámetros con el cuerpo de la función. Es el símbolo que le da nombre.
  • Retorno Implícito: Si la función tiene una sola línea, no necesita llaves {} ni la palabra return. JavaScript entiende que debe devolver ese resultado automáticamente.

Diferencias de Sintaxis

Dependiendo de la complejidad, la función flecha puede verse de distintas formas:

// 1. Sin parámetros (usa paréntesis vacíos)
const saludo = () => "Hola mundo";

// 2. Un solo parámetro (sin paréntesis)
const cuadrado = (x) => x * x;

// 3. Múltiples parámetros (con paréntesis)
const suma = (a, b) => a + b;

// 4. Con varias líneas de código (necesita llaves y return)
const login = (usuario) => {
    console.log("Iniciando sesión...");
    return "Bienvenido " + usuario;
};

¿Qué es un Arreglo (Array)?

Es una estructura de datos que permite almacenar varios valores en una sola variable. Cada elemento tiene una posición llamada índice, que siempre comienza en 0.

let frutas = ["Manzana", "Pera", "Banana"];
// Índice:       0          1         2

1. Recorrer un Arreglo (Bucle For)

Para leer todos los elementos de un arreglo, usamos un bucle for. La propiedad .length nos dice cuántos elementos tiene el arreglo para saber cuándo detenerse.

let colores = ["Rojo", "Verde", "Azul"];

for (let i = 0; i < colores.length; i++) {
    console.log("Color: " + colores[i]);
}

2. Adicionar Elementos (.push)

Para agregar un nuevo elemento al final del arreglo, utilizamos el método .push().

let alumnos = ["Juan", "Maria"];
alumnos.push("Pedro"); // Agrega al final

console.log(alumnos); // ["Juan", "Maria", "Pedro"]

3. Editar Elementos

Para modificar un valor, simplemente accedemos a su índice y le asignamos un nuevo dato, igual que con una variable normal.

let compras = ["Pan", "Leche", "Huevos"];

// Queremos cambiar "Leche" (índice 1) por "Yogur"
compras[1] = "Yogur";

console.log(compras); // ["Pan", "Yogur", "Huevos"]

4. Eliminar Elementos (.splice)

El método .splice() permite eliminar elementos en cualquier posición. Recibe dos números: el índice inicial y cuántos elementos quieres borrar. 

let juegos = ["Mario", "Zelda", "Metroid", "Sonic"];

// Eliminar "Zelda" (está en el índice 1)
juegos.splice(1, 1); 

console.log(juegos); // ["Mario", "Metroid", "Sonic"]

¿Qué es un Objeto?

A diferencia de un arreglo que es una lista, un objeto es una colección de propiedades. Cada propiedad tiene una clave (nombre) y un valor. Se usan para representar entidades del mundo real.

const usuario = {
    nombre: "Dario",
    edad: 30,
    profesion: "Ingeniero",
    esPremium: true
};

Acceso a Propiedades

Para leer el valor de una propiedad, usamos la notación de punto (la más común) o la notación de corchetes (necesaria si la clave tiene espacios). 

console.log(usuario.nombre);    // "Dario"
console.log(usuario["edad"]);    // 30

Modificar y Agregar

Los objetos son flexibles. Puedes cambiar un valor existente o crear una propiedad nueva simplemente asignándole un valor.

// Modificar
usuario.edad = 31;

// Agregar nueva propiedad
usuario.pais = "Bolivia";

console.log(usuario.pais); // "Bolivia"

Métodos (Funciones)

Un objeto también puede realizar acciones. Cuando una propiedad es una función, se le llama método. 

const perro = {
    nombre: "Firulais",
    ladrar: function() {
        return "¡Guau guau!";
    },
    saludar() {
        return `Hola, soy ${this.nombre}`;
    }
};

console.log(perro.saludar()); // "Hola, soy Firulais"

Objetos Anidados

Un objeto puede contener otros objetos o arreglos dentro, permitiendo estructuras de datos más complejas. 

const curso = {
    titulo: "JavaScript",
    autor: {
        nombre: "Dario",
        web: "aprende365"
    },
    temas: ["Variables", "Funciones", "Objetos"]
};

console.log(curso.autor.web); // "aprende365"

¿Qué es el DOM?

DOM significa Document Object Model (Modelo de Objetos del Documento). Es una estructura en forma de árbol que representa tu HTML. JavaScript usa el DOM para leer, cambiar o borrar elementos de la página en tiempo real.

// El DOM convierte tu HTML en un "Objeto" que JS entiende.
console.log(document); // Muestra todo el HTML de la página

Seleccionar Elementos

Antes de cambiar algo, debemos "atrapar" el elemento. Las formas más comunes son por su ID o usando selectores de tipo CSS.

// Por ID (el más rápido)
const titulo = document.getElementById("mi-titulo");

// Por selector de CSS (muy versátil)
const boton = document.querySelector(".btn-principal");
const parrafos = document.querySelectorAll("p"); // Atrapa TODOS los p

Modificar Contenido y Estilo

Una vez seleccionado, podemos cambiar su texto, su HTML interno o incluso sus estilos de CSS directamente desde JavaScript.

const caja = document.querySelector("#mensaje");

// Cambiar el texto
caja.textContent = "¡Hola desde el DOM!";

// Cambiar el HTML (permite etiquetas)
caja.innerHTML = "<b>Texto en negrita</b>";

// Cambiar CSS
caja.style.backgroundColor = "blue";
caja.style.color = "white";

Eventos (Interactividad)

Los eventos son acciones que ocurren en la página (clics, teclas presionadas, mover el ratón). Usamos addEventListener para "escuchar" cuando el usuario hace algo. 

const btn = document.querySelector("#mi-boton");

// Escuchar el clic
btn.addEventListener("click", () => {
    alert("¡Hiciste clic en el botón!");
    document.body.style.backgroundColor = "yellow";
});

Reto: Controlador de Flechas

Podemos usar un switch o varios if para detectar qué flecha de dirección presionó el usuario. ¡Esto es la base para mover personajes en un juego!

let usuarios = [

    { id: 2, nombre: "Ana Lopez", email: "ana@ejemplo.com" },

];

const filasPorPagina = 2;

// 2. Función para renderizar la tabla
function renderizarTabla(lista) {
    const tbody = document.querySelector("#tbodyUsuarios");
    tbody.innerHTML = "";

    // Lógica de paginación
    let inicio = (paginaActual - 1) * filasPorPagina;
    let fin = inicio + filasPorPagina;
    let paginados = lista.slice(inicio, fin);

    paginados.forEach(u => {
        tbody.innerHTML += `
            <tr>
                <td>${u.id}</td>
                <td>${u.nombre}</td>
                <td>${u.email}</td>
                <td><button class="btn-delete" onclick="eliminarUsuario(${u.id})">Eliminar</button></td>
            </tr>
        `;
    });
    
    document.querySelector("#infoPagina").textContent = `Página ${paginaActual}`;
}

// 3. Eliminar con confirmación
function eliminarUsuario(id) {
    if (confirm("¿Estás seguro de eliminar este usuario?")) {
        usuarios = usuarios.filter(u => u.id !== id);
        renderizarTabla(usuarios);
    }
}

document.querySelector("#buscador").addEventListener("input", (e) => {

        u.email.toLowerCase().includes(termino)

    paginaActual = 1; // Reiniciar a la primera página al buscar
    renderizarTabla(filtrados);
});

// 5. Adicionar Usuario (Ejemplo rápido)

        // Solo si el ancho de pantalla es menor a 768px (Móvil)
        if (window.innerWidth < 768) {
            e.stopPropagation(); // Evita que el evento suba al padre

            const submenu = this.querySelector("ul");
            
            if (submenu) {
                // Alternamos una clase para mostrar/ocultar
                submenu.style.display = submenu.style.display === "block" ? "none" : "block";
            }
        }
    });
});

Lógica del Slider

// 1. Arreglo de objetos con los datos de las imágenes
const slides = [
    { url: "https://picsum.photos/id/10/600/350", texto: "Montañas Nevadas" },
    { url: "https://picsum.photos/id/20/600/350", texto: "Escritorio de Trabajo" },
    { url: "https://picsum.photos/id/30/600/350", texto: "Taza de Café" }
];

let indiceActual = 0;

// 2. Función para actualizar la imagen en el DOM
function mostrarSlide(index) {
    const img = document.querySelector("#slider-img");
    const caption = document.querySelector("#slider-caption");

    // Validar límites (efecto infinito)
    if (index >= slides.length) indiceActual = 0;
    else if (index < 0) indiceActual = slides.length - 1;
    else indiceActual = index;

    // Cambiar contenido
    img.src = slides[indiceActual].url;
    caption.textContent = slides[indiceActual].texto;
}

// 3. Función para los botones
function cambiarSlide(direccion) {
    mostrarSlide(indiceActual + direccion);
}

// 4. Autoplay (Opcional)
setInterval(() => {
    cambiarSlide(1);
}, 5000); // Cambia cada 5 segundos

// Inicializar
mostrarSlide(indiceActual);
