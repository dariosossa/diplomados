I. Fundamentos de CSS

El lenguaje estructurado para la presentación y diseño de interfaces web.

CSS (Cascading Style Sheets o Hojas de Estilo en Cascada) es el lenguaje utilizado para definir la presentación visual de un documento estructurado en HTML. Controla elementos como la tipografía, los colores, el espaciado, la disposición de los elementos y su adaptabilidad a diferentes dispositivos.

1. Sintaxis y Archivos

CSS opera mediante Reglas. Una regla se compone de un Selector (el elemento objetivo) y un bloque de Declaraciones delimitado por llaves (las propiedades a modificar).

Estructura de Sintaxis CSS
/* Selector */
p {
  /* Declaración: Propiedad y Valor */
  color: blue;
  font-size: 18px;
}

2. Vinculación de CSS con HTML

La práctica estándar y recomendada para el desarrollo web es utilizar hojas de estilo externas. Esto permite separar la estructura (HTML) de la presentación (CSS). Se vincula mediante la etiqueta <link> en el <head> del documento HTML.

3. Estructura HTML Base

El siguiente bloque de código representa la estructura básica de un documento HTML5, configurado para integrar una hoja de estilos externa.

index.html (Estructura Base) 
<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Prácticas de CSS</title>
    
    <!-- Vinculación de la hoja de estilos externa -->
    <link rel="stylesheet" href="css/style.css">
</head>
<body>

    <!-- Contenido principal del documento -->
    <h1>Módulo de CSS Aplicado</h1>

</body>
</html>

4. Reset CSS (Normalización de estilos)

Los navegadores web aplican márgenes y rellenos predeterminados a diversos elementos HTML. Para asegurar consistencia en el diseño a través de diferentes navegadores, se utiliza una regla de "Reset Universal".

css/style.css (Reset Universal) 
/* Reset de propiedades predeterminadas */
* {
    margin: 0;
    padding: 0;
    box-sizing: border-box; /* Fundamental para el cálculo de dimensiones */
}

5. Arquitectura del Modelo de Caja (Box Model)

El principio fundamental del diseño web radica en que todo elemento HTML se renderiza como una caja rectangular. Esta caja está compuesta por cuatro áreas concéntricas:

1. MARGIN (Margen Exterior)
2. BORDER (Borde Físico)
3. PADDING (Espacio Interno)
4. CONTENT

6. Selectores de Estructura: ID y Class

HTML (Definición de Selectores)
<!-- Atributo ID (#): Define un identificador exclusivo por documento. -->
<div id="contenedor-principal">Bloque Principal Exclusivo</div>

<!-- Atributo CLASS (.): Define una clase reutilizable para múltiples elementos. -->
<div class="bloque-estandar">Bloque Genérico 1</div>
<div class="bloque-estandar">Bloque Genérico 2</div>
CSS (Reglas para Selectores)
/* Selector ID (Utiliza el prefijo #) */
#contenedor-principal {
    margin: 40px;
    padding: 30px;
    border: 4px solid;
}

/* Selector de Clase (Utiliza el prefijo .) */
.bloque-estandar {
    margin: 15px;
    padding: 10px;
    border: 2px dashed;
}

7. Gestión del Espaciado: Margin y Padding

  • Padding: Define el espacio entre el contenido del elemento y su propio borde.
  • Margin: Define el espacio exterior del elemento, separándolo de otros elementos adyacentes en el flujo del documento.
HTML
<div class="elemento-espaciado">
    Elemento con propiedades de espaciado aplicadas.
</div>

<div class="elemento-centrado">
    Elemento centrado horizontalmente.
</div>
CSS
.elemento-espaciado {
    background-color: #64748b;
    /* PADDING: Asigna 20px de espacio interno en los 4 lados */
    padding: 20px; 
    /* MARGIN: Superior(10px) Derecho(20px) Inferior(30px) Izquierdo(40px) */
    margin: 10px 20px 30px 40px; 
}

.elemento-centrado {
    background-color: #3b82f6;
    color: white;
    width: 300px;
    padding: 20px;
    /* TÉCNICA DE CENTRADO: El valor 'auto' en los márgenes laterales 
       calcula el espacio disponible y centra el elemento en su contenedor. */
    margin: 50px auto; 
}

8. Propiedades Estructurales de Bordes

La delimitación visual de una caja se define mediante tres propiedades fundamentales de borde: Grosor (width), Estilo (style) y Color.

HTML
<div class="borde-estandar">Elemento con borde delimitado</div>
CSS
.borde-estandar {
  /* Declaración agrupada: Grosor | Estilo | Color */
  border: 3px solid black;
  padding: 20px;
}

9. Variaciones de Estilo de Borde

El comportamiento de trazado del borde puede modificarse para distintos propósitos de interfaz:

  • solid: Trazado continuo estandar.
  • dashed: Trazado discontinuo mediante guiones (útil para zonas de arrastrar y soltar).
  • dotted: Trazado de puntos continuos.
  • double: Trazado de línea paralela doble.

10. Sistemas de Representación de Color

  • Hexadecimal (#): Representación alfanumérica de 6 caracteres del modelo RGB. Ej: #ff0000 (Rojo base). Es el formato predominante en la industria.
  • RGB: Representación decimal de valores de Rojo, Verde y Azul (0 a 255). Ej: rgb(0, 255, 0).
  • RGBA: Añade el canal Alfa (Alpha) para gestionar la opacidad. Ej: rgba(0, 0, 255, 0.5) (Azul con 50% de opacidad).

14. Dimensiones Relativas y Funciones de Cálculo (vw, vh, calc)

El diseño adaptativo requiere unidades de medida relativas que respondan a las dimensiones del entorno de visualización.

  • % (Porcentaje): Dimensión calculada en proporción al área de su contenedor principal directo.
  • vw (Viewport Width): Dimensión calculada en relación al 100% de la anchura total de la pantalla.
  • vh (Viewport Height): Dimensión calculada en relación al 100% de la altura de la pantalla.
  • calc(): Función CSS nativa para resolver operaciones aritméticas de dimensionamiento de manera dinámica.
CSS
.ancho-viewport { width: 100vw; background: #f59e0b; }
.alto-viewport { height: 50vh; background: #10b981; }

.contenedor-calculado {
    /* Ocupa el espacio vertical disponible deduciendo 100 píxeles */
    height: calc(100vh - 100px);
    background: #8b5cf6;
}

17. Imágenes de Fondo y Mapeo Object-Fit

Para incluir contenido gráfico dentro del modelo de caja manteniendo proporciones estructurales, se utiliza `background-image` o la etiqueta `` junto a la propiedad `object-fit`.

HTML
<div class="banner-cabecera">
    <h1>Estructura Visual</h1>
</div>

<img src="https://images.unsplash.com/photo-1464822759023-fed622ff2c3b" alt="Paisaje" class="imagen-adaptada">
CSS
/* Método A: Capa de Fondo CSS */
.banner-cabecera {
    width: 100%; 
    height: 300px;
    background-image: url("https://images.unsplash.com/photo-1464822759023-fed622ff2c3b");
    /* Ajusta la imagen a los límites del contenedor garantizando la cobertura total */
    background-size: cover; 
    background-position: center; 
    color: white;
}

/* Método B: Elemento Imagen HTML */
.imagen-adaptada {
    width: 300px;
    height: 300px;
    /* Previene la distorsión del aspecto ratio original al forzar las dimensiones */
    object-fit: cover; 
    border-radius: 10px;
}

18. Escala Tipográfica (Unidades Rem)

Para asegurar un diseño escalable en las fuentes tipográficas se emplea la unidad rem, la cual escala en relación al valor raíz del documento (habitualmente 16px).

CSS

.encabezado-primario { font-size: 2.5rem; } /* Equivale a 40px (2.5 x 16) */

21. La Propiedad Display: Modificación de la Naturaleza del Renderizado

Define las reglas de comportamiento del elemento con respecto al flujo del documento DOM.

  • block: Demanda la totalidad del ancho horizontal del contenedor. Propicia un salto de línea en su finalización. Permite establecer dimensiones específicas.
  • inline: El elemento adapta su área estrictamente al tamaño del contenido. Se posiciona en la misma línea que el elemento precedente. Inhabilita las propiedades width/height.
  • inline-block: Combina el flujo secuencial horizontal de tipo inline, pero habilita las propiedades de dimensionamiento de tipo block.
CSS
.forzar-flujo-bloque { display: block; background: #ef4444; }

.forzar-flujo-linea { display: inline; background: #3b82f6; color: white;}

.hibrido-dimensional { 
    display: inline-block; 
    width: 250px; /* Dimensionamiento procesado correctamente */
    background: #10b981;
}

22. Ocultamiento de Elementos del DOM

CSS

.preservar-espacio { 
    /* El elemento pierde su representación visual pero el DOM reserva su espacio original. */

}

26. Animaciones Interactivas (Transition y Hover)

La regla de transición procesa las variaciones de propiedades a través de interpolación temporal progresiva, evitando actualizaciones abruptas al aplicar pseudo-clases dinámicas de eventos como :hover.

CSS
.elemento-animado {
    background-color: #3b82f6;
    color: white;
    padding: 15px 30px;
    border-radius: 8px;
    
    /* DECLARACIÓN DE CÁLCULO PROGRESIVO: Atributos Afectados | Duración | Curva Bézier */
    transition: all 0.3s ease; 
}

/* Pseudo-Clase Activadora: Accionada cuando el sensor del cursor intercepta la caja */
.elemento-animado:hover {
    background-color: #1e3a8a;
    /* Alteración estructural: Modifica la escala vectorial al 110% de manera animada */
    transform: scale(1.1); 
    box-shadow: 0px 8px 15px rgba(0,0,0,0.2);
}

27. Media Queries y Adaptación Responsiva

Reglas lógicas condicionales. Permiten inyectar secuencias especiales de código CSS sobre la página tras realizar una consulta y validación con los valores de las dimensiones de hardware de la pantalla del equipo origen.

La metodología Mobile First consiste en determinar el código estructural base a la resolución mínima. Evaluaciones de pantallas superiores van sobrescribiendo estos cimientos.

CSS (Lógica Condicional Mobile-First)
/* 1. ESTRUCTURA BASE PREESTABLECIDA (Aplicada a resoluciones bajas - Móvil) */
.caja-evaluada { 
    background-color: #ef4444; 
    width: 100%; 
} 

/* 2. MEDIA QUERY - CONDICIONAL (Disparo activo al superar los 768 píxeles - Desktop) */
@media (min-width: 768px) {
    .caja-evaluada { 
        background-color: #10b981; /* Sobrescribe la regla de color */
        width: 50%; /* Sobrescribe y limita la regla de ancho total */
    } 
}

II. Flexbox: Motor Vectorial Estructural

Mecanismo estandarizado para alineación en sistemas de diseño unidimensional.

Flexbox (Flexible Box Layout) constituye el motor nativo primario de estructuración CSS desarrollado para distribuir grupos y contenedores sobre el flujo unilineal (una distribución en eje X u eje Y).

28. Contenedores y Configuración de Dirección de Flujo

El requerimiento técnico esencial inicial exige definir una etiqueta contenedora padre e inicializarla con el comando display: flex;. Esta acción activa las funciones interactivas para reubicación de sus descendientes.

HTML
<!-- Elemento Principal y Agrupador Vectorial -->
<div class="agrupador-flex">
    <div class="modulo-hijo">1</div>
    <div class="modulo-hijo">2</div>
    <div class="modulo-hijo">3</div>
</div>
CSS
.agrupador-flex {
    display: flex; /* Iniciación del motor estructural */
    
    /* FLEX-DIRECTION: Configura el sentido del flujo lineal.
       'row': Fila de base horizontal (Izquierda-Derecha). Valor Standard.
       'column': Columna base vertical (Arriba-Abajo). Eje crítico en Mobile Layout. */
    flex-direction: row; 
}

29. Propiedad Justify Content (Alineación en Eje Primario X)

Regula la separación de elementos y su anclaje sobre el eje de flujo que dicta Flexbox.

CSS (Control de Desplazamientos Horizontales)
.barra-horizontal-flex {
    display: flex;
    flex-direction: row;
    
    /* JUSTIFY-CONTENT:
       'flex-start': Condensación de componentes al sector originario de partida.
       'flex-end': Condensación total al sector final del bloque delimitado.
       'center': Centralización aglutinada equitativamente en medio.
       'space-between': Envía a los bordes y crea un espacio único vacío de por medio.
       'space-around': Separa en bloques homogéneos mediante márgenes parejos. */
       
    justify-content: space-between; 
}

31. Propiedad Flex Wrap (Gestión de Elementos Excedentes)

En el flujo predeterminado flex, las cajas internas son presionadas unas con otras para que calcen juntas al modificarse sus dimensiones. Para evitarlo utilizamos flex-wrap, el cual rompe la línea cuando superan los bordes y crea arreglos ordenados consecutivos inferiores.

CSS
.catalogo-adaptativo {
    display: flex;
    /* 'wrap': Ordena a todos los elementos que excedan su área, continuar 
               la serie reubicándose en la siguiente fila inferior dinámica. */
    flex-wrap: wrap; 
}

32. Generación Inteligente de Brechas Inter-Elementos (Gap)

Las propiedades primigenias de margin añaden distancias alrededor y afueras del contenedor global, lo que ensucia cálculos estéticos. Gap genera un espaciado vacío "únicamente" en el centro o separación existente entre entidades Flex anexadas.

CSS
.galeria-ordenada {
    display: flex;
    flex-wrap: wrap;
    /* Configura las áreas sin contacto exactamente como un canal de 20px */
    gap: 20px; 
}

Actividad 30: PROYECTO FINAL (El Desafío Maestro)

El Proyecto Final: Desarrollo de un Sistema Web Completo

Demuestra tu dominio de CSS diseñando las interfaces completas para un sistema web real. La temática es libre (ej. Tienda E-Commerce, Software Clínico, Inventario Estudiantil, Plataforma de Envios).

Deberás estructurar tu proyecto creando cuatro páginas HTML interconectadas, aplicando obligatoriamente Diseño Responsivo (Mobile-First) a todas ellas:

1. Página de Aterrizaje (index.html):
Construye la portada pública de tu sistema. Debe incluir una Navbar Pública pegajosa (Sticky) con algún menú dropdown oculto, un Banner Fotográfico (Hero Image) con capa superpuesta de filtro oscuro, y una cuadrícula controlada en Flexbox con 3 Cards Verticales que detallen los servicios primarios del negocio.

2. Página de Inicio de Sesión (login.html):
Diseña una interfaz de pantalla dividida (Split Screen). Un lado estará protagonizado por una imagen representativa y publicitaria, y el otro estará provisto por el formulario principal organizado, enfocado y alineado perfectamente al centro del área.

3. Panel de Administrador (admin.html):
Crea el panel de control del sistema. Aplica el Layout centralizado con Sidebar fija a la izquierda y Navbar tipo WebApp en formato oscuro. En el área principal (área de trabajo remanente de scroll aislado), coloca un bloque con tarjetas de métricas numéricas. En el inferior, diseña íntegramente una Tabla CRUD Flexbox para gestionar componentes de bases de datos. Dicha tabla debe mutar obligatoriamente y reformularse como tarjetas apiladas dinámicas si la ventana se reduce al formato móvil.

4. Panel de Usuario Común (usuario.html):
Diseña la interfaz secundaria para el registro y actualización del usuario final. Genera implementaciones de Pestañas (Tabs) interactivas utilizando JS simple para poder intercalar secciones o bloques descriptivos diferentes, y haz uso de tus formularios CSS previamente formulados a dos columnas dinámicas dentro del panel.
