Manual Práctico: DOM y Git/GitHub

Este manual web sirve como referencia rápida para la manipulación del Document Object Model mediante JavaScript puro (Vanilla JS) y el uso avanzado de Git y GitHub para control de versiones.

PARTE 1: El DOM (Document Object Model)

1. ¿Qué es el DOM?

El DOM es una interfaz de programación para documentos HTML. Representa la página como un árbol de nodos (elementos padre, hijo, hermano) permitiendo que JavaScript cambie la estructura, estilo y contenido en tiempo real.

2. Seleccionar Elementos

El primer paso para manipular la web es "capturar" los elementos HTML.

// 1. Por ID (Único)
const titulo = document.getElementById('mi-titulo');

// 2. Por Clase o Etiqueta (Devuelven colecciones HTML)
const botones = document.getElementsByClassName('btn');
const parrafos = document.getElementsByTagName('p');

// 3. Selectores CSS (La forma MODERNA)
const primerBoton = document.querySelector('.btn'); // Devuelve solo el primero
const todosLosBotones = document.querySelectorAll('.btn'); // Devuelve todos (NodeList)

3. Modificar Elementos

Cambia el contenido, los estilos en línea o las clases de CSS.

const caja = document.querySelector('#mi-caja');

// --- Contenido ---
caja.textContent = 'Texto seguro sin HTML'; 
caja.innerHTML = '<strong>Texto en negrita</strong>'; // Usa con cuidado (riesgo XSS)

// --- Estilos CSS (usar camelCase) ---
caja.style.backgroundColor = '#2563eb';
caja.style.display = 'none';

// --- Clases CSS ---
caja.classList.add('activa');    // Añadir
caja.classList.remove('oculta'); // Quitar
caja.classList.toggle('modo-oscuro'); // Alternar (si está la quita, si no, la pone)

4. Crear y Eliminar Elementos

// --- Crear ---
// 1. Crear nodo
const nuevoParrafo = document.createElement('p');
// 2. Modificarlo
nuevoParrafo.textContent = 'Soy un párrafo dinámico';
nuevoParrafo.classList.add('destacado');
// 3. Inyectarlo en el DOM
const contenedor = document.querySelector('#contenedor');
contenedor.appendChild(nuevoParrafo); // Lo pone al final
// contenedor.prepend(nuevoParrafo); // Lo pone al principio

// --- Eliminar ---
const elementoViejo = document.querySelector('#viejo');
elementoViejo.remove();

5. Eventos (Interactividad)

Detectar clicks, teclado, formularios, etc.

const boton = document.querySelector('#btn-guardar');
const form = document.querySelector('#mi-formulario');

// Evento de Click
boton.addEventListener('click', function(evento) {
    console.log('¡Me hiciste click!');
    console.log('Elemento clickeado:', evento.target);
});

// Evento de Formulario
form.addEventListener('submit', function(evento) {
    evento.preventDefault(); // Evita que la página se recargue
    alert('Formulario procesado con JS');
});

PARTE 2: Git y GitHub

1. Conceptos Básicos

  • Git: Motor local de control de versiones.
  • GitHub: Servidor web que aloja tus repositorios de Git y facilita la colaboración.

2. Instalación de Git

Antes de usar Git, necesitas instalarlo en tu computadora:

  • Windows: Descarga el instalador desde git-scm.com y ejecuta el archivo .exe (puedes dejar todas las opciones que vienen marcadas por defecto).
  • macOS: Abre la terminal y escribe git --version. Si no lo tienes, te aparecerá una ventana pidiendo instalar las herramientas de línea de comandos de Apple. También puedes instalarlo usando Homebrew: brew install git.
  • Linux (Ubuntu/Debian): Abre tu terminal y ejecuta el comando sudo apt install git.
# Para comprobar que se instaló correctamente, abre tu terminal y escribe:
git --version

3. Crear cuenta en GitHub

Para guardar tus proyectos en la nube y colaborar, necesitas una cuenta gratuita:

  1. Ve a github.com.
  2. Haz clic en el botón "Sign up" (Registrarse) arriba a la derecha.
  3. Ingresa tu correo electrónico, crea una contraseña segura y elige tu nombre de usuario.
  4. Resuelve el puzzle de seguridad y verifica tu cuenta con el código enviado a tu correo.

4. Configuración Inicial

git config --global user.name "Tu Nombre"
git config --global user.email "tu@email.com"
git config --global init.defaultBranch main # Establecer main como rama por defecto

5. Flujo de Trabajo Básico

git init                  # Iniciar repositorio
git status                # Ver estado de los archivos
git add index.html        # Añadir un archivo específico al stage
git add .                 # Añadir TODOS los archivos modificados
git commit -m "Mensaje"   # Guardar los cambios con un mensaje claro

6. Ramas (Branches)

git branch                # Listar ramas locales
git switch nueva-rama     # Moverse a una rama existente
git switch -c fix-menu    # Crear y moverse a una rama nueva
git merge fix-menu        # Fusionar rama 'fix-menu' en tu rama actual

7. GitHub (Remoto)

git remote add origin https://github.com/usuario/repo.git # Vincular
git push -u origin main   # Subir por primera vez
git push                  # Subir cambios (después de la primera vez)
git pull                  # Bajar los cambios del servidor
git clone https://...     # Clonar un repositorio completo

8. Conectar con Visual Studio Code (Fácil)

VS Code tiene una excelente integración visual con Git y GitHub que te permite evitar escribir comandos en la terminal para las tareas cotidianas:

  1. Abre la carpeta de tu proyecto en VS Code.
  2. Ve al panel de Control de código fuente en la barra lateral izquierda (el icono que parece un árbol de ramificaciones o presiona Ctrl+Shift+G).
  3. Si tu proyecto no tiene Git aún, haz clic en el botón "Initialize Repository" (equivale a usar git init).
  4. Para guardar cambios, escribe un mensaje en la caja de texto superior y haz clic en el botón "Commit" (acepta si te pregunta si deseas añadir todos los archivos al 'stage').
  5. Para enviar el código a GitHub, haz clic en el botón "Publish Branch" o "Publish to GitHub".
  6. El paso de conexión: La primera vez que hagas esto, VS Code te pedirá permiso para acceder a GitHub. Se abrirá una ventana en tu navegador web. Simplemente haz clic en "Autorizar" y la conexión estará hecha.
  7. ¡Listo! A partir de ahora podrás usar el botón "Sync Changes" (Sincronizar cambios) para hacer push y pull de manera automática.

9. Ignorar Archivos (.gitignore) ⭐ (Avanzado)

El archivo .gitignore es vital en cualquier proyecto. Se crea en la raíz del proyecto y le dice a Git qué archivos jamás debe rastrear ni subir (ej. carpetas muy pesadas, contraseñas, configuraciones personales).

¿Cómo crearlo? Simplemente crea un archivo de texto llamado exactamente .gitignore (con el punto al inicio).

Patrones de escritura en .gitignore:

# --- CARPETAS ---
node_modules/       # Ignora toda la carpeta node_modules (vital en JS)
dist/               # Ignora la carpeta de compilación
build/

# --- ARCHIVOS ESPECÍFICOS ---
.env                # ¡MUY IMPORTANTE! Ignora variables de entorno/contraseñas
config.json         # Archivo específico
database.sqlite     # Bases de datos locales

# --- POR EXTENSIÓN ---
*.log               # Ignora todos los archivos que terminen en .log
*.zip               # Ignora todos los zips
*.DS_Store          # Archivos de sistema de Mac

# --- EXCEPCIONES (El símbolo !) ---
*.log               # Ignora todos los .log
!produccion.log     # EXCEPTO este archivo en particular

# --- RUTAS ESPECÍFICAS ---
logs/*.txt          # Ignora los .txt solo dentro de la carpeta logs/
¡Problema Común! "He añadido el archivo al .gitignore pero Git lo sigue subiendo"
Esto ocurre porque Git ya había registrado ese archivo antes de que lo pusieras en el .gitignore. Para que Git "olvide" los archivos y aplique el .gitignore correctamente, debes limpiar la caché: 
# 1. Quitar todos los archivos de la caché de Git (no los borra de tu disco duro)
git rm -r --cached .

# 2. Volver a añadir todo (ahora Git respetará el .gitignore)
git add .

# 3. Hacer un commit para guardar el estado limpio
git commit -m "Limpiar caché de archivos y aplicar .gitignore"

10. Deshacer Errores

# Ver historial de commits
git log --oneline

# Modificar el mensaje del ÚLTIMO commit o añadirle archivos que olvidaste
git commit --amend -m "Nuevo mensaje corregido"

# Revertir un archivo a como estaba en el último commit (Pierdes cambios no guardados)
git restore nombre-archivo.html

# Eliminar un commit (Peligroso si ya hiciste push)
git reset --hard HEAD~1   # Borra el commit y los cambios en el código
git reset --soft HEAD~1   # Borra el commit pero mantiene tu código intacto en el editor

PARTE 3: Ejercicios Prácticos (Paso a Paso)

Sigue estas instrucciones detalladas para poner en práctica lo aprendido. Crea una carpeta nueva en tu computadora llamada practica-curso para empezar.

Retos de DOM

  1. El Botón Mágico (Eventos básicos):
    • Paso 1: Crea un archivo index.html. Añade un botón con id="btn-magico" y un párrafo vacío con id="mensaje".
    • Paso 2: Abre una etiqueta <script> al final de tu HTML (antes de cerrar el body).
    • Paso 3: Usa document.getElementById() en JavaScript para capturar el botón y el párrafo. Guárdalos en variables.
    • Paso 4: Añádele un evento al botón: boton.addEventListener('click', function() { ... }).
    • Paso 5: Dentro de la función, haz que el texto del párrafo cambie: parrafo.textContent = "¡Hola, DOM!";

  2. Modo Oscuro (Modificar clases):
    • Paso 1: En tu HTML, añade una etiqueta <style> en el head y crea una clase CSS: .tema-oscuro { background-color: #333; color: white; }
    • Paso 2: Añade un nuevo botón en tu HTML con id="btn-tema".
    • Paso 3: En JavaScript, selecciona ese botón y selecciona toda la página usando const cuerpo = document.body;
    • Paso 4: Añade el evento click al botón y dentro usa: cuerpo.classList.toggle('tema-oscuro');. ¡Haz clic varias veces para ver la magia!

  3. Lista Dinámica (Crear elementos - Avanzado):
    • Paso 1: Crea un <input id="tarea">, un <button id="btn-agregar"> y una lista vacía <ul id="lista"></ul>.
    • Paso 2: En JS, selecciona los tres elementos.
    • Paso 3: Al hacer click en el botón, crea un "li" nuevo usando: const nuevoLi = document.createElement('li');
    • Paso 4: Ponle texto al "li" extrayendo el valor del input: nuevoLi.textContent = input.value;
    • Paso 5: Insértalo en la página: lista.appendChild(nuevoLi);

Retos de Git y GitHub

  1. Tu primer Repo (Flujo local):
    Abre la terminal en la carpeta donde creaste tus HTML y ejecuta estos comandos en orden:
    • git init (Inicia el rastreo en la carpeta).
    • git status (Verás tu index.html en rojo).
    • git add . (Prepara todos los archivos para guardarlos).
    • git commit -m "Mi primer código de DOM terminado" (Guarda la versión).

  2. El Archivo Oculto (.gitignore):
    • Paso 1: Crea un archivo en tu carpeta llamado privado.txt y escribe un secreto dentro.
    • Paso 2: Crea otro archivo llamado exactamente .gitignore
    • Paso 3: Abre el archivo .gitignore, escribe dentro la palabra privado.txt y guarda.
    • Paso 4: Ejecuta git status. Verás que Git detecta el .gitignore, ¡pero ignora por completo la existencia de tu archivo privado!

  3. Viaje Multiverso (Ramas y Fusión):
    • Paso 1: Crea una línea temporal alternativa: git switch -c nueva-cabecera
    • Paso 2: Abre tu HTML, ponle un <h1>Mi Súper Página</h1> al principio y guarda el archivo.
    • Paso 3: Guarda los cambios en Git: git add . seguido de git commit -m "Añadido título principal".
    • Paso 4: Viaja al pasado/línea original: git switch main. (¡Mira tu código en el editor, el H1 desapareció mágicamente!)
    • Paso 5: Trae el código del futuro al presente: git merge nueva-cabecera. (¡El H1 volvió!)
