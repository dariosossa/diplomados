I. Preparando tu Entorno de Trabajo

El lienzo en blanco: HTML y JavaScript unidos.

Todo proyecto web profesional separa su estructura visual (HTML/CSS) de su cerebro lógico (JavaScript). Para empezar, necesitas crear dos archivos en la misma carpeta de tu computadora.

1. Creando index.html y script.js

Crea un archivo llamado index.html. En él, pondremos la estructura básica y, justo antes de cerrar la etiqueta </body>, llamaremos a nuestro archivo de lógica.

index.html (El cascarón)
<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <title>Mi Primera App en JS</title>
</head>
<body>
    <h1>Bienvenido al Curso de JavaScript</h1>
    <p>Abre la consola de tu navegador para ver la magia.</p>
    
    <!-- VINCULACIÓN CRÍTICA: Llamamos al archivo externo -->
    <script src="script.js"></script>
</body>
</html>

Ahora, crea un archivo llamado script.js en esa misma carpeta. Aquí es donde escribiremos todo nuestro código a partir de ahora.

II. Memoria y Lógica: Variables y Condicionales

Para que una aplicación funcione, necesita "recordar" datos y tomar "decisiones" basadas en esos datos.

2. Variables: Las cajas de memoria

Usamos const para cajas selladas (valores que no cambiarán, como tu fecha de nacimiento). Usamos let para cajas abiertas (valores que cambiarán, como la edad o el saldo).

script.js (Variables)
// TIPOS DE DATOS BÁSICOS

// 1. String (Texto, siempre entre comillas)
const nombreUsuario = "Juan Pérez"; 

// 2. Number (Números para matemáticas, sin comillas)
let edad = 17; 

// 3. Boolean (Verdadero o Falso)
let tieneSuscripcion = true;

console.log("Usuario:", nombreUsuario);
console.log("Edad inicial:", edad);

// JS permite reasignar variables creadas con 'let'
edad = 18; 
console.log("Edad actualizada:", edad);

3. Condicionales (If / Else)

Las computadoras no piensan, solo siguen reglas. El bloque if (Si pasa esto) y else (Sino, haz esto otro) es la base de la inteligencia artificial y de cualquier sistema.

script.js (Tomando Decisiones)
let saldo = 50;
let costoProducto = 100;

// Preguntamos: ¿Es el saldo mayor o igual al costo?
if (saldo >= costoProducto) {
    console.log("¡Compra exitosa! Tienes dinero suficiente.");
    saldo = saldo - costoProducto; // Restamos el dinero
} else {
    console.log("Fondos insuficientes. Te faltan $" + (costoProducto - saldo));
}

III. Agrupando Datos: Objetos y Arrays

Las variables simples guardan un solo dato. Pero en una tienda real, un producto tiene nombre, precio, stock y categoría. Para agrupar todo eso usamos Objetos y Arreglos.

4. Objetos (Fichas de Identidad)

Un objeto se crea con llaves { }. Sirve para describir las propiedades de una sola cosa.

script.js (Objetos)
const celular = {
    marca: "Apple",
    modelo: "iPhone 15",
    precio: 999,
    enStock: true
};

// Para acceder a una característica específica, usamos el punto (.)
console.log( "El celular es marca:", celular.marca );
console.log( "Su precio es de $", celular.precio );

5. Arreglos (Listas de Elementos)

Un Arreglo (Array) se crea con corchetes [ ]. Es una lista ordenada. Lo más poderoso de JavaScript es combinar ambas cosas: Un Arreglo que contiene muchos Objetos. ¡Así es como se ven exactamente las Bases de Datos!

script.js (Array de Objetos)
// Una lista [ ] que adentro tiene múltiples objetos { }
const inventario = [
    { id: 1, nombre: "Laptop HP", precio: 1200 },
    { id: 2, nombre: "Mouse Gamer", precio: 45 },
    { id: 3, nombre: "Teclado Mecánico", precio: 80 }
];

// ¡OJO! Los arreglos en programación empiezan a contar desde el CERO (0)
// El Mouse Gamer es el elemento número 1.
console.log( "El segundo producto es:", inventario[1].nombre ); 
console.log( "El precio del tercer producto es:", inventario[2].precio );

IV. Funciones y Procesamiento de Datos

Ya tenemos nuestra base de datos. Ahora necesitamos procesarla. ¿Qué pasa si quiero mostrar el nombre de todos los productos? No puedo escribir console.log 100 veces. Necesito un bucle.

6. El Bucle "forEach" (Por cada uno)

El método .forEach() viaja a través del arreglo, toma un elemento a la vez, y ejecuta el código que le digamos.

script.js (Recorriendo Arreglos)
const listaTareas = ["Lavar ropa", "Estudiar JS", "Hacer compras"];

// Le decimos al arreglo: "Por cada (forEach) tarea que tengas adentro, haz esto:"
// Usamos la sintaxis de 'Función Flecha' () => {}
listaTareas.forEach( (tareaActual) => {
    console.log("Tengo pendiente:", tareaActual);
});

7. Buscando Datos con .filter()

Para hacer buscadores no destruimos el arreglo original. Usamos .filter(). Este método revisa el arreglo, y si la condición es verdadera (true), guarda ese elemento en un arreglo nuevo.

script.js (Filtrando Información)
const tienda = [
    { nombre: "TV Samsung", precio: 500, categoria: "Electrónica" },
    { nombre: "Silla de Oficina", precio: 80, categoria: "Muebles" },
    { nombre: "Laptop Dell", precio: 900, categoria: "Electrónica" }
];

// Queremos SOLO los productos de electrónica
const productosElectronicos = tienda.filter( (prod) => {
    return prod.categoria === "Electrónica"; // Si es igual, lo guarda. Si no, lo descarta.
});

console.log("Filtro aplicado:", productosElectronicos);

V. El DOM: El Puente hacia el HTML

Hasta ahora, JavaScript solo ha hablado en secreto con la Consola. ¡Queremos que interactúe con el usuario en la pantalla real! Para eso usamos el DOM (Document Object Model).

El DOM es una herramienta interna del navegador que convierte todo tu archivo HTML en "nodos" manipulables. JavaScript usa 3 pasos para controlar la página: 1) Atrapar, 2) Escuchar, 3) Modificar.

8. Eventos y Modificación en vivo

HTML & script.js (Interactividad Básica)
<!-- EN TU INDEX.HTML -->
<h1 id="titulo-app">Sistema Desconectado</h1>
<button id="btn-conectar">Conectar Ahora</button>

<!-- EN TU SCRIPT.JS -->
// PASO 1: ATRAPAR (Buscamos los elementos por su ID usando el '#')
const titulo = document.querySelector("#titulo-app");
const boton = document.querySelector("#btn-conectar");

// PASO 2: ESCUCHAR (Añadimos un EventListener esperando un 'click')
boton.addEventListener("click", () => {
    
    // PASO 3: MODIFICAR EL DOM
    titulo.textContent = "¡Sistema Conectado Exitosamente!"; // Cambia el texto
    titulo.style.color = "green"; // Cambia el color CSS
    
    // ¡También podemos ocultar el botón una vez clickeado!
    boton.style.display = "none"; 
});

VI. Inyectando HTML desde JavaScript

En el mundo real, los desarrolladores no escriben 100 etiquetas <li> a mano en el HTML para mostrar 100 productos. El servidor nos envía el Arreglo de datos, y nosotros usamos el bucle forEach para fabricar y pintar las etiquetas automáticamente.

9. El poder de innerHTML y los Template Literals

Para esto usaremos innerHTML (que inyecta código HTML dentro de un contenedor) y unas comillas especiales llamadas Template Literals (` `), que nos permiten meter variables dentro de texto fácilmente usando ${ }.

HTML & JS (Fabricación de Interfaz)
<!-- EN TU HTML (Dejamos una caja vacía esperando los datos) -->
<h2>Nuestros Productos:</h2>
<ul id="caja-productos"></ul>

<!-- EN TU SCRIPT.JS -->
const catalogo = [
    { id: 1, nombre: "Zapatillas Deportivas", precio: 55 },
    { id: 2, nombre: "Camiseta de Algodón", precio: 20 },
    { id: 3, nombre: "Gorra Urbana", precio: 15 }
];

// Atrapamos la caja 
     vacía
const cajaUl = document.querySelector("#caja-productos");

// Recorremos el arreglo de productos
catalogo.forEach((producto) => {
    
    // += significa "Súmale esto al final de lo que ya tenías"
    // Las comillas invertidas ` ` nos permiten armar HTML multilinea
    cajaUl.innerHTML += `
        <li style="margin-bottom: 10px; border-bottom: 1px solid gray;">
            <b>Producto:</b> ${producto.nombre} <br>
            <i>Cuesta:</i> $${producto.precio}
        </li>
    `;
    
});

VII. Arquitectura de "Estado"

🧠 Cambio de mentalidad crítico: A medida que añadimos botones para "Borrar" o un "Buscador", si empezamos a borrar y añadir pedazos de HTML sueltos, nuestro código será un desastre inmanejable.

La programación moderna dicta una regla inquebrantable: Separar los Datos de la Vista.
1. Tendremos un ESTADO (Una gran variable que guarda todos los arreglos y textos de búsqueda).
2. Tendremos UNA SOLA FUNCIÓN (Render) que lee ese Estado, limpia toda la pantalla y la pinta de nuevo de golpe.
3. Cuando el usuario hace clic en "Borrar", nosotros NO borramos el HTML. Nosotros borramos el dato del Arreglo (El Estado), y luego mandamos a llamar a la función Render para que repinte todo.

VIII. Construyendo Componentes Reales

Vamos a unir el DOM, los Eventos y esta nueva forma de pensar para construir elementos avanzados.

10. Menús de Navegación Desplegables (Toggle)

Un menú oculto en realidad siempre está ahí en el HTML, solo tiene una clase de CSS que lo esconde (ej. display: none u .oculto). La función classList.toggle() hace la magia de poner o quitar esa clase con un solo clic.

HTML & JS (Acordeón Sidebar)
<!-- TU HTML -->
<ul>
    <li>
        <button class="btn-acordeon">Opciones de Sistema ▾</button>
        <!-- Esta caja tiene 'oculto' (display: none) en el archivo CSS -->
        <ul class="sub-menu oculto">
            <li><a href="#">Mi Perfil</a></li>
            <li><a href="#">Configuración</a></li>
        </ul>
    </li>
</ul>

<!-- TU SCRIPT.JS -->
// Atrapamos todos los botones que abren menús
const botonesAcordeon = document.querySelectorAll('.btn-acordeon');

botonesAcordeon.forEach(boton => {
    boton.addEventListener('click', function() {
        
        // 'this' es una palabra especial que significa "El botón específico que acabas de clickear"
        // nextElementSibling atrapa la etiqueta que está inmediatamente debajo en el HTML (el <ul> sub-menu)
        const sublista = this.nextElementSibling;
        
        // toggle: Si tiene la clase 'oculto', se la quita. Si no, se la pone.
        sublista.classList.toggle('oculto');
        
        // Detalle UX: Cambiar la flechita
        if (sublista.classList.contains('oculto')) {
            this.innerHTML = this.innerHTML.replace('▴', '▾');
        } else {
            this.innerHTML = this.innerHTML.replace('▾', '▴');
        }
    });
});

11. El Componente Maestro: La Tabla CRUD

Aquí es donde graduarás tus habilidades. Usaremos el Objeto de Estado, la inyección con innerHTML y el método .filter() para el buscador y para borrar. Si entiendes este bloque, estás listo para el desarrollo web avanzado.

HTML (El Cascarón de la Tabla)
<div style="margin-bottom: 15px;">
    <input type="text" id="buscador" placeholder="Buscar empleado...">
</div>

<table border="1" width="100%">
    <thead>
        <tr>
            <th>ID</th><th>Nombre</th><th>Acción</th>
        </tr>
    </thead>
    <!-- La magia de JS inyectará las filas iteradas aquí -->
    <tbody id="cuerpo-tabla"></tbody>
</table>
script.js (Motor de Estado y Renderizado)
// 1. EL ESTADO (La única fuente de la verdad de toda la app)
let estado = {
    empleados: [
        { id: 1, nombre: "Ana López" },
        { id: 2, nombre: "Carlos Ruiz" },
        { id: 3, nombre: "Beatriz Moreno" }
    ],
    textoBuscado: "" // Guardará lo que escribamos en el input
};

const cuerpoTabla = document.getElementById('cuerpo-tabla');
const inputBuscador = document.getElementById('buscador');

// 2. FUNCIÓN DE RENDERIZADO (Se encarga de leer el estado y pintar todo)
function renderizarTabla() {
    
    // A. Filtramos el arreglo. Crea una copia temporal para mostrar
    const datosAVisualizar = estado.empleados.filter( item => 
        item.nombre.toLowerCase().includes(estado.textoBuscado)
    );

    // B. Limpiamos la tabla vieja borrando todo su HTML interno
    cuerpoTabla.innerHTML = ''; 
    
    // C. Pintamos la nueva información
    datosAVisualizar.forEach(item => {
        cuerpoTabla.innerHTML += `
            <tr>
                <td>${item.id}</td>
                <td>${item.nombre}</td>
                <!-- OJO: Le incrustamos el ID directamente en el botón para saber cuál borrar -->
                <td><button onclick="eliminarRegistro(${item.id})">Borrar</button></td>
            </tr>
        `;
    });
}

// 3. EVENTO DE BÚSQUEDA
inputBuscador.addEventListener('input', (evento) => {
    // Al escribir, actualizamos la variable del estado
    estado.textoBuscado = evento.target.value.toLowerCase();
    // Y mandamos a repintar (La función se encarga de todo el trabajo pesado)
    renderizarTabla(); 
});

// 4. FUNCIÓN DE ELIMINACIÓN
// Tiene 'window.' para que el botón HTML la encuentre sin problemas
window.eliminarRegistro = (idClick) => {
    // Confirmación de seguridad
    if (confirm("¿Seguro que deseas borrar este registro?")) {
        
        // Reemplazamos el arreglo original por uno nuevo que NO contenga el ID clickeado
        estado.empleados = estado.empleados.filter(item => item.id !== idClick);
        
        // Repintamos la tabla con el arreglo ya modificado
        renderizarTabla(); 
    }
};

// 5. ARRANQUE DEL SISTEMA
// Llamamos a la función una vez al principio para que pinte la pantalla por primera vez
renderizarTabla();

IX. Conectando con el Mundo Real: APIs y Fetch

Comunicación con bases de datos remotas.

Hasta ahora, todos nuestros datos (el arreglo de empleados) estaban guardados en la memoria de nuestro navegador. Si recargas la página, cualquier cambio se pierde. En el mundo real, los datos viven en un servidor (Backend/Base de Datos).

Para comunicarnos con el servidor, usamos una API (Application Programming Interface). JavaScript se comunica con las APIs usando una función llamada fetch y los verbos HTTP (GET para leer, POST para guardar, DELETE para borrar).

12. Asincronismo: Promesas y async/await

Ir a buscar datos a un servidor en internet toma tiempo (milisegundos o segundos). JavaScript es impaciente y no le gusta esperar. Si le decimos que busque datos, pasará a la siguiente línea de código inmediatamente, arruinando nuestra tabla porque los datos aún no llegaron.

Para obligar a JS a esperar, usamos dos palabras mágicas: async (para avisar que la función toma tiempo) y await (para decirle "detente aquí y espera la respuesta").

13. CRUD Real con una API Pública

Vamos a reescribir nuestra Tabla CRUD usando JSONPlaceholder, una API pública gratuita perfecta para pruebas. (Nota: JSONPlaceholder simula los POST y DELETE para no dañar su servidor público, te responderá que fue un éxito, pero no guardará el cambio real en su base de datos).

HTML (Tabla con indicador de carga)
<!-- Agregamos un indicador visual mientras JS espera al servidor -->
<h3 id="mensaje-cargando" style="color: #38bdf8;">Conectando al servidor... Espere.</h3>

<table border="1" width="100%">
    <thead>
        <tr>
            <th>ID</th><th>Nombre (Desde la API)</th><th>Acción</th>
        </tr>
    </thead>
    <tbody id="cuerpo-tabla-api"></tbody>
</table>
script.js (Fetch API + Estado)
// 1. EL ESTADO AHORA EMPIEZA VACÍO
let estadoAPI = {
    usuarios: [],     // Esperando datos del servidor
    cargando: true    // Controla el mensaje de "Espere..."
};

const URL_API = 'https://jsonplaceholder.typicode.com/users';
const tablaAPI = document.getElementById('cuerpo-tabla-api');
const msjCarga = document.getElementById('mensaje-cargando');

// ==========================================
// FUNCIÓN RENDER (La misma de siempre)
// ==========================================
function renderizarTablaAPI() {
    // Si está cargando, mostramos el mensaje y salimos
    if (estadoAPI.cargando) {
        msjCarga.style.display = 'block';
        tablaAPI.innerHTML = '';
        return;
    }

    // Si ya cargó, ocultamos el mensaje y pintamos
    msjCarga.style.display = 'none';
    tablaAPI.innerHTML = ''; 

    estadoAPI.usuarios.forEach(user => {
        tablaAPI.innerHTML += `
            <tr>
                <td>${user.id}</td>
                <td>${user.name}</td>
                <td><button onclick="borrarUsuarioAPI(${user.id})">Eliminar en Servidor</button></td>
            </tr>
        `;
    });
}

// ==========================================
// READ (GET) - OBTENER DATOS DE LA API
// ==========================================
// Usamos 'async' para poder usar 'await' adentro
async function obtenerUsuarios() {
    estadoAPI.cargando = true;
    renderizarTablaAPI(); // Mostramos el loader

    try {
        // 1. AWAIT: "JS, espera a que el servidor responda"
        const respuesta = await fetch(URL_API);
        
        // 2. AWAIT: "JS, espera a convertir esa respuesta en un Arreglo utilizable (JSON)"
        const datosReales = await respuesta.json();
        
        // 3. Guardamos los datos reales en nuestro ESTADO
        estadoAPI.usuarios = datosReales;
        
    } catch (error) {
        console.error("Hubo un error contactando a la API:", error);
    }

    // 4. Apagamos el loader y pintamos los datos
    estadoAPI.cargando = false;
    renderizarTablaAPI();
}

// ==========================================
// DELETE - BORRAR DATOS EN LA API
// ==========================================
window.borrarUsuarioAPI = async (idBorrar) => {
    if (confirm("¿Borrar definitivamente de la Base de Datos?")) {
        
        try {
            // Mandamos la orden al servidor (Method: 'DELETE')
            // Le pasamos el ID al final de la URL: /users/5
            await fetch(`${URL_API}/${idBorrar}`, {
                method: 'DELETE',
            });
            
            // Si el servidor no dio error, actualizamos nuestro estado local
            estadoAPI.usuarios = estadoAPI.usuarios.filter(user => user.id !== idBorrar);
            
            // Repintamos
            renderizarTablaAPI();
            alert("Usuario eliminado en el servidor.");

        } catch (error) {
            alert("Error al borrar en el servidor.");
        }
    }
};

// ==========================================
// INICIO DE LA APP
// ==========================================
obtenerUsuarios();
