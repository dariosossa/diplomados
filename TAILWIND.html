MÓDULO TAILWIND CSS • DESDE CERO A COMPONENTES

I. Tailwind CSS: Utility-First

El adiós a los archivos CSS externos interminables.

Tailwind CSS revoluciona el desarrollo web eliminando la necesidad de escribir hojas de estilo semánticas (ej. .tarjeta-principal) en archivos separados. En su lugar, utilizamos clases de utilidad directamente en las etiquetas de nuestro HTML.

1. Instalación Rápida (CDN)

Para aprender y practicar en archivos locales sin instalar herramientas complejas (como Node.js), utilizamos su enlace CDN. Solo debes pegarlo una vez en tu archivo HTML.

HTML: Vinculación de Tailwind
<!-- Pega esto dentro de la etiqueta <head> de tu archivo HTML -->
<script src="https://cdn.tailwindcss.com"></script>

II. Fundamentos: Traduciendo CSS Clásico

Antes de crear componentes gigantes, debemos aprender cómo "habla" Tailwind. Cada propiedad que conoces en CSS tiene una clase equivalente, corta y lógica.

2. Colores de Fondo y Texto

Tailwind trae una paleta predefinida numerada del 50 (muy claro) al 900 (muy oscuro). El prefijo bg- es para el fondo (Background) y text- para las letras.

Colores en Tailwind
/* EN CSS CLÁSICO HACÍAMOS ESTO:
.alerta-error {
    background-color: #ef4444; 
    color: #ffffff;
}
*/
<!-- EN TAILWIND SE HACE ASÍ DIRECTAMENTE: -->
<div class="bg-red-500 text-white">
  Ha ocurrido un error crítico.
</div>

<!-- Otro ejemplo: Fondo azul claro, texto azul oscuro -->
<div class="bg-blue-100 text-blue-900">
  Información guardada correctamente.
</div>
Actividad 1: Pinta tu primera caja

Práctica Rápida

2. Crea un <div> y añádele las clases para que tenga un fondo verde oscuro (green-800) y letras en verde muy clarito (green-100).

3. Espaciado: Margin y Padding

En Tailwind, el espaciado usa un sistema de cuartos de rem (1 unidad = 4px).
Padding (Interior): Usamos la letra p. (ej. p-4, px-2, pt-5).
Margin (Exterior): Usamos la letra m. (ej. m-4, my-2, ml-5).

  • t = Top (Arriba)
  • b = Bottom (Abajo)
  • l = Left (Izquierda)
  • r = Right (Derecha)
  • x = Eje X (Izquierda y Derecha)
  • y = Eje Y (Arriba y Abajo)
Margin y Padding en Tailwind
/* EN CSS CLÁSICO HACÍAMOS ESTO:
.caja {
    padding: 16px;            /* Padding a todos lados */
    padding-left: 32px;       /* Padding izquierdo grande */
    margin-top: 20px;         /* Margen superior */
    margin-bottom: 20px;      /* Margen inferior */
}
*/
<!-- EN TAILWIND SE HACE ASÍ: -->
<div class="p-4 pl-8 my-5 bg-slate-200">
  Caja con espaciado perfecto.
</div>

4. Tipografía y Textos

Tailwind nos permite cambiar el tamaño, el grosor (weight) y la alineación usando prefijos lógicos.

Textos en Tailwind
/* EN CSS CLÁSICO HACÍAMOS ESTO:
h1 {
    font-size: 24px;      
    font-weight: 700;     /* Negrita */
    text-align: center;   /* Centrado */
    text-transform: uppercase; /* Mayúsculas */
}
*/
<!-- EN TAILWIND SE HACE ASÍ: -->
<h1 class="text-2xl font-bold text-center uppercase">
  Título Principal
</h1>

<!-- Tamaños: text-xs, text-sm, text-base, text-lg, text-xl, text-2xl, text-4xl... -->
<!-- Grosor: font-light, font-normal, font-semibold, font-bold, font-extrabold -->
Actividad 2: Estructurando un Párrafo

Práctica Rápida

Crea un <p> en tu HTML. Aplícale Tailwind para que el texto sea grande (lg), de color gris oscuro (gray-700), alineado a la derecha (right) y tenga un relleno general equivalente a 16px (4 unidades).

5. Bordes, Redondeado y Sombras (Efecto 3D)

El aspecto visual moderno ("Tarjetas") se logra redondeando esquinas y aplicando sombras sutiles que separan elementos del fondo.

Bordes y Sombras en Tailwind
/* EN CSS CLÁSICO HACÍAMOS ESTO:
.tarjeta-basica {
    border: 1px solid #e2e8f0;
    border-radius: 8px;
    box-shadow: 0 4px 6px -1px rgba(0, 0, 0, 0.1);
}
*/
<!-- EN TAILWIND SE HACE ASÍ: -->
<div class="border border-slate-200 rounded-lg shadow-md p-6 bg-white">
  Esto ya parece una tarjeta profesional.
</div>

<!-- Redondeados: rounded-sm, rounded, rounded-lg, rounded-xl, rounded-full (circulos) -->
<!-- Sombras: shadow-sm, shadow, shadow-md, shadow-lg, shadow-xl -->

6. Flexbox Básico en Tailwind

Activar Flexbox es tan sencillo como poner la clase flex. Luego usas sus utilidades para alinear (items-...) y justificar (justify-...).

Flexbox en Tailwind
/* EN CSS CLÁSICO HACÍAMOS ESTO:
.caja-centrada {
    display: flex;
    justify-content: center;
    align-items: center;
    gap: 16px;
}
*/
<!-- EN TAILWIND SE HACE ASÍ: -->
<div class="flex justify-center items-center gap-4 bg-slate-100 p-4">
  <div class="bg-blue-500 w-10 h-10 rounded-full"></div>
  <div class="bg-red-500 w-10 h-10 rounded-full"></div>
</div>
Actividad 3: La Barra de Navegación Básica

Práctica Rápida

Usa Flexbox en Tailwind para simular un Menú. Crea un contendor oscuro. Usa flex y justify-between. A la izquierda coloca el nombre de tu empresa (ej. LOGO) y a la derecha crea un botón que diga "Login". Aplica items-center para que no queden chuecos.

III. Biblioteca de Componentes UI Completos

¡Ahora sí! Teniendo las bases claras (colores, espaciado, bordes y flexbox), vamos a combinarlas todas juntas para crear los componentes que necesitas en tu sistema final.

7. Botones Interactivos y Hover

Agregamos interactividad. En Tailwind usamos el prefijo hover: antes de cualquier clase para indicar qué debe pasar al pasar el mouse por encima.

Botón Interactivo
/* EN CSS CLÁSICO:
 .btn { background: #2563eb; color: white; padding: 12px 24px; border-radius: 8px; transition: 0.3s; }
 .btn:hover { background: #1d4ed8; transform: translateY(-2px); }
*/
<!-- EN TAILWIND: -->
<button class="bg-blue-600 text-white font-semibold py-3 px-6 rounded-lg transition-all duration-300 hover:bg-blue-700 hover:-translate-y-1 shadow-md hover:shadow-lg">
  Confirmar Acción
</button>

10. Pantalla Dividida - Split Screen (Para login.html)

Aquí usamos Media Queries de Tailwind. El prefijo md: significa "solo aplícate en pantallas medianas o más grandes (Tablets/PCs)". Por defecto (celulares), es una columna (flex-col).

Login Split Screen Responsivo
/* EN CSS CLÁSICO:
 .layout { display: flex; height: 100vh; flex-direction: column; }
 @media (min-width: 768px) { .layout { flex-direction: row; } }
 .mitad-img { display: none; }
 @media (min-width: 768px) { .mitad-img { display: flex; flex: 1; } }
*/
<!-- EN TAILWIND: -->
<div class="flex h-screen w-full flex-col md:flex-row">
  
  <!-- LADO IZQUIERDO: Imagen (hidden en móvil, flex al 50% en desktop) -->
  <div class="hidden md:flex md:w-1/2 bg-slate-900 items-center justify-center bg-[url('https://images.unsplash.com/photo-1544197150-b99a580bb7a8')] bg-cover relative">
    <div class="absolute inset-0 bg-indigo-900/60 mix-blend-multiply"></div>
  </div>

  <!-- LADO DERECHO: Formulario (100% en móvil, 50% en desktop) -->
  <div class="w-full md:w-1/2 flex items-center justify-center p-6 bg-slate-50">
    <form class="w-full max-w-md bg-white p-8 sm:p-10 rounded-2xl shadow-xl border border-slate-100">
      <h2 class="text-3xl font-extrabold mb-8 text-slate-800 text-center">Bienvenido</h2>
      
      <div class="mb-5">
        <label class="block text-slate-700 text-sm font-bold mb-2">Correo Electrónico</label>
        <input type="email" class="w-full px-4 py-3 rounded-lg bg-slate-50 border border-slate-200 focus:border-blue-500 focus:bg-white focus:ring-2 outline-none transition">
      </div>
      
      <button class="w-full bg-blue-600 text-white font-bold py-3 rounded-lg hover:bg-blue-700 transition">Entrar</button>
    </form>
  </div>

</div>

11. Tabla CRUD Responsiva (Para admin.html)

Usando el mismo prefijo md:, hacemos que la tabla se muestre como bloques apilados (tarjetas) en el celular, y vuelva a ser fila de tabla (md:table-row) en escritorio.

Tabla Mutante (Móvil a Escritorio)
/* EN CSS CLÁSICO:
 @media (max-width: 768px) {
  .tabla-crud thead { display: none; }
  .tabla-crud tr { display: block; border: 1px solid #ccc; margin-bottom: 10px; border-radius: 8px; }
  .tabla-crud td { display: block; padding: 10px; text-align: right; }
 }
*/
<!-- EN TAILWIND: -->
<div class="overflow-x-auto bg-white rounded-xl shadow-md p-4 md:p-0 border border-slate-100">
  <table class="w-full text-left border-collapse">
    
    <!-- CABECERA: Oculta en móvil -->
    <thead class="hidden md:table-header-group bg-slate-800 text-white uppercase text-xs font-semibold tracking-wider">
      <tr>
        <th class="p-4 rounded-tl-lg">ID</th>
        <th class="p-4">Usuario</th>
        <th class="p-4 rounded-tr-lg text-right">Acciones</th>
      </tr>
    </thead>
    
    <!-- CUERPO -->
    <tbody class="block md:table-row-group">
      
      <!-- FILA 1: 'block' en móvil (parece tarjeta), 'table-row' en pc -->
      <tr class="block md:table-row border border-slate-200 md:border-b md:border-x-0 hover:bg-slate-50 mb-4 md:mb-0 rounded-lg md:rounded-none p-3 md:p-0 transition">
        
        <td class="p-2 md:p-4 block md:table-cell text-sm font-bold text-slate-500">
          <span class="md:hidden text-xs text-slate-400 font-normal uppercase">ID: </span>#101
        </td>
        
        <td class="p-2 md:p-4 block md:table-cell text-blue-600 font-semibold">
          <span class="md:hidden text-xs text-slate-400 font-normal uppercase block">Usuario:</span>
          Rubén Darío
        </td>
        
        <td class="p-2 md:p-4 block md:table-cell md:text-right mt-2 md:mt-0">
          <button class="bg-red-50 text-red-600 px-4 py-2 rounded-md text-sm font-medium hover:bg-red-100 w-full md:w-auto transition">Eliminar</button>

EL PROYECTO FINAL (Aprobación del Módulo)

Desafío: Sistema Web con Tailwind CSS

Demuestra tu dominio estructurando las 4 interfaces de un sistema web real (Temática Libre) utilizando exclusivamente las clases de Tailwind CSS enseñadas arriba:

1. index.html (Página Pública): Crea el componente "Hero Banner" superpuesto y debajo implementa una grilla Flexbox con al menos 3 Tarjetas de Servicio (Cards).

2. login.html (Ingreso al Sistema): Crea el formulario centrado utilizando el patrón de Pantalla Dividida (Split Screen responsivo a columnas en móviles).

3. admin.html (Panel de Administración): Implementa el Dashboard Layout completo (Sidebar + Header + Área de trabajo). Dentro del área de trabajo, coloca la Tabla CRUD Responsiva.

4. usuario.html (Panel de Usuario): Reutiliza el Dashboard Layout. En el área central, crea formularios sencillos combinando Flexbox y las Inputs del login.

💡 Instrucción Crítica: Para que los comandos funcionen, asegúrate de colocar el CDN <script src="https://cdn.tailwindcss.com"></script> en la etiqueta <head> de tus 4 archivos HTML. ¡No necesitas crear ningún archivo .css!
